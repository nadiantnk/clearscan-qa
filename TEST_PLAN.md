# Тест-план: ClearScan

**Проект:** ClearScan — приложение записи пациентов  
**Версия документа:** 1.0  
**Автор:** Надя Антоненко  
**Дата:** 25.03.2026  
**Статус:** Утверждён

---

## 1. Введение

Документ описывает объём, подход и ресурсы для тестирования веб-приложения ClearScan. Приложение предназначено для регистраторов диагностического центра и используется для ведения расписания приёма пациентов на КЛКТ/ОПТГ-исследования.

Тестирование проводится в рамках формирования QA-портфолио. Приложение представляет собой single-page application без серверной части — все данные хранятся в localStorage браузера.

---

## 2. Цели тестирования

- убедиться, что основные пользовательские сценарии работают корректно
- проверить валидацию всех полей формы записи
- выявить дефекты в логике расчёта стоимости (скидки)
- проверить корректность отчётов и финансовых итогов
- убедиться, что граничные значения обрабатываются правильно

---

## 4. Что НЕ входит в scope

- нагрузочное тестирование (нет сервера)
- тестирование безопасности
- тестирование на мобильных устройствах (приложение не адаптировано)
- тестирование совместимости с IE / Safari < 15

---

## 5. Виды тестирования

**Функциональное тестирование** — основной вид. Проверяется соответствие поведения приложения ожидаемому результату.

**Граничные значения** — проверка полей с ограничениями: скидка 1–100%, длина телефона, формат email, дата рождения.

**Негативное тестирование** — ввод некорректных данных, проверка реакции системы.

**Проверка UI** — соответствие визуального состояния логике (подсветка статусов, активность кнопки «Сохранить»).

**Регрессионное тестирование** — после исправления найденных дефектов.

---

## 6. Критерии начала тестирования

- приложение открывается в браузере без ошибок в консоли
- все разделы навигации доступны
- форма создания записи открывается

---

## 7. Критерии завершения тестирования

- все тест-кейсы из плана выполнены
- все найденные дефекты задокументированы в BUG_REPORTS.md
- критических (блокирующих) дефектов не осталось незафиксированными
