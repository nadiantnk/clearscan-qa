### BUG-001 — [Расписание] После заполнения карточки, верстка смещается вправо

**Severity:** Medium  
**Priority:** Low  
**Статус:** Open 
**Окружение:** Firefox 143.0, Windows 11  

**Описание:**  
При сохранении новой карточки данные пациента отображаются некорректно, поля "Телефон" "Клиника", "Услуги", "Статус" смещаются вправо

**Предусловие**
Выполнен вход с правами администратора

**Шаги для воспроизведения:**
1. Навести курсор на любую строку в колонке "ФИО пациента"
2. Нажать "Создать запись"
3. Внести необходимые данные
4. Нажать "Сохранить"

**Ожидаемый результат:**  
При сохранении карточки, интерфейс стабилен, элементы не смещаются

**Фактический результат:**  
После нажатия «Сохранить» происходит горизонтальный сдвиг всей таблицы вправо.

### BUG-002 - [Расписание] [Новая запись] Поле "Дата рождения" принимает недопустимые значения

**Severity:** Major  
**Priority:** Medium 
**Статус:** Open 
**Окружение:** Firefox 143.0, Windows 11  

**Описание:**  
Поле "Дата рождения" пропускает значения 1590

**Предусловие**
Выполнен вход с правами администратора, нажата кнопка "Создать запись" и заполены все обязательные поля валидными данными

**Шаги для воспроизведения:**
1. Ввести в поле "Дата рождения" 01.01.1590
2. Нажать "Сохранить"

**Ожидаемый результат:**  
Система не позволяет сохранить запись и отображается сообщение об ошибке

**Фактический результат:**  
Запись успешно сохраняется с невалидными данными

### BUG-003 - [Новая запись] В поле "Скидка" отсутствует валидация

**Severity:** Minor
**Priority:** Medium 
**Статус:** Open 
**Окружение:** Firefox 143.0, Windows 11 

**Описание:** 
Поле "Скидка" принимает значения "/", "-", "+", "*"


**Предусловие**
Выполнен вход с правами администратора, нажата кнопка "Создать запись", заполнены все обязательные поля валидными данными

**Шаги для воспроизведения:**
1. В поле скидка ввести один из символов "-", "+", "/", "*"
2. Нажать "Сохранить"

**Ожидаемый результат:**  
Система не позволяет сохранить запись и отображается сообщение об ошибке

**Фактический результат:**   
Запись успешно сохраняется с невалидными данными. При повторном открытии записи введённый символ отсутствует, скидка не применена, сумма к оплате не изменилась

### BUG-004 — [Расписание] Кнопка «Перейти на сегодня» в календаре разбита на три строки

**Severity:** Minor
**Priority:** Low
**Статус:** Open 
**Окружение:** Firefox 143.0, Windows 11 

**Описание:** 
Текст кнопки «Перейти на сегодня» в виджете календаря отображается в три строки вместо одной. 

**Предусловие** 
Выполнен вход с правами администратора

**Шаги для воспроизведения:**
1. Открыть раздел "Расписание"
2. Нажать на дату в правом верхнем углу («вс, 29 марта ∨»)
3. В выпадающем виджете наблюдать кнопку "Перейти на сегодня"

**Ожидаемый результат:**  
Текст кнопки "Перейти на сегодня" отображается в одну строку

**Фактический результат:**  
Текст кнопки "Перейти на сегодня" разбит на три строки
