### BUG-001 — [Расписание] После заполнения карточки, верстка смещается вправо

**Severity:** Medium  
**Priority:** Low  
**Статус:** Open 
**Окружение:** Firefox 143.0, Windows 11  

**Описание:**  
При сохранении новой карточки данные пациента отображаются некорректно, поля "Телефон" "Клиника", "Услуги", "Статус" смещаются вправо

**Предусловие**
Выполнен вход с правами администратора

**Шаги для воспроизведения:**
1. Навести курсор на любую строку в колонке "ФИО пациента"
2. Нажать "Создать запись"
3. Внести необходимые данные
4. Нажать "Сохранить"

**Ожидаемый результат:**  
При сохранении карточки, интерфейс стабилен, элементы не смещаются

**Фактический результат:**  
После нажатия «Сохранить» происходит горизонтальный сдвиг всей таблицы вправо.

### BUG-002 - [Расписание] [Новая запись] В поле "Дата рождения" отстуствует валидация

**Severity:** Medium  
**Priority:** High 
**Статус:** Open 
**Окружение:** Firefox 143.0, Windows 11  

**Описание:**  
Поле "Дата рождения" пропускает значения 1590 и 2222г

**Предусловие**
Выполнен вход с правами администратора, нажата кнопка "Создать запись" и внесены все обязательные поля валиными данными

**Шаги для воспроизведения:**
1. Ввести в поле "Дата рождения" 01.01.1590
2. Нажать "Сохранить"
3. Ввести в поле "Дата рождения" 01.01.2222
4. Нажать "Сохранить"

**Ожидаемый результат:**  
Система выдает ошибку "Некорректная дата" 

**Фактический результат:**  
Запись успешно сохраняется с невалидными данными
